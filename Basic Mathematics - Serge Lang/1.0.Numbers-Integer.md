# ஆசிரியர் உண்மையில் சொல்ல விரும்புவது

எண்களை இப்படி மனதில் கற்பனை செய்ய வேண்டும்:

```text
Number = Position
Addition = Movement
Positive = Right
Negative = Left
Zero = Origin
-a = Opposite Side of a
```

உதாரணமாக,

```text
8 + (-12)
```

என்று பார்த்தால்,

"8-இல் இருந்து 12 unit இடது பக்கம் செல்"

என்று உடனே மனதில் தோன்ற வேண்டும்.

```text
8 → 7 → 6 → 5 → 4 → 3 → 2 → 1 → 0 → -1 → -2 → -3 → -4
```

அதனால்

```text
8 + (-12) = -4
```

இதுதான் இந்த Section-ன் முழு Geometry intuition. ஆசிரியர் Numbers-ஐ வெறும் குறியீடுகளாக அல்ல, **Number Line-ல் உள்ள இடங்களாகவும் நகர்வுகளாகவும்** பார்க்க பயிற்சி கொடுக்கிறார்.

பதில்: **இரண்டும் கொஞ்சம் உண்மை.**

### 1. Actual Math என்ன?

கடுமையான (formal) கணிதத்தில், "3 என்பது ஒரு இடம்" அல்லது "5 + (-3) என்பது இடது பக்கம் நகர்வது" என்று வரையறுக்கப்படவில்லை.

கணிதவியலாளர்கள் முதலில் சில அடிப்படை விதிகளை (axioms) ஏற்றுக்கொள்கிறார்கள்:

* 0 என்ற எண் உள்ளது.
* ஒவ்வொரு எண்ணுக்கும் ஒரு எதிர் எண் (inverse) உள்ளது.
* கூட்டல் associative.
* கூட்டல் commutative.

போன்ற விதிகள்.

அதிலிருந்து மெதுவாக எல்லா முடிவுகளையும் நிரூபிக்கிறார்கள்.

---

### 2. அப்படியென்றால் Number Line ஏன்?

Number Line என்பது **ஒரு மாதிரி (model)** அல்லது **ஒரு காட்சி விளக்கம் (visual interpretation)**.

உதாரணம்:

```text
5 + (-3) = 2
```

என்பதை

```text
5-இல் இருந்து 3 unit இடது பக்கம் செல்
```

என்று நினைத்தால் எளிதாக புரிகிறது.

ஆனால் அது கணிதத்தின் அடிப்படை வரையறை அல்ல.

---

### 3. ஆசிரியர் ஏன் இப்படி கற்பிக்கிறார்?

இந்த புத்தக ஆசிரியர் (Serge Lang) ஒரு முக்கியமான கருத்தை நம்புகிறார்:

> முதலில் intuition (உள்ளுணர்வு) வர வேண்டும்; பிறகு formal proof வர வேண்டும்.

அதனால் அவர் Number Line, Thermometer, Distance போன்ற உதாரணங்களை அதிகமாக பயன்படுத்துகிறார். 

---

### 4. ஆழமாகப் பார்த்தால்?

சுவாரஸ்யமான விஷயம்:

நீ நினைப்பது போல "Actual Math" மற்றும் "Number Line Model" இரண்டும் முற்றிலும் வேறல்ல.

உண்மையில் Integers-க்கான விதிகள்:

```text
a + (-a) = 0
```

என்பதையும்

Number Line-ல்:

```text

```

0-வின் எதிரெதிர் பக்கங்களில் இருப்பதையும்

ஒரே அமைப்பு (structure) பிரதிபலிக்கிறது.

அதனால் Number Line வெறும் கற்பனை அல்ல; அது Integers-ன் அமைப்பை மிகவும் துல்லியமாக பிரதிநிதித்துவப்படுத்துகிறது.

* ஆனால் Number Line Integers எப்படி நடந்து கொள்கின்றன என்பதை மிகவும் துல்லியமாக காட்டுகிறது.

---

எனவே சுருக்கமாக:

> **"5 + (-3) என்பது 5-இல் இருந்து 3 unit இடது பக்கம் செல்வது" என்பது ஆசிரியர் கொடுத்த ஒரு intuition மட்டும் அல்ல; அது Integers-ன் உண்மையான அமைப்பை பிரதிபலிக்கும் ஒரு சரியான mathematical model. ஆனால் formal mathematics-ல் கூட்டல் அதைப் பயன்படுத்தி வரையறுக்கப்படுவதில்லை; axioms-ஐ பயன்படுத்தி வரையறுக்கப்படுகிறது.**


------------------------------------------


`a + (-a) = 0, ஆமா இதை எப்படி புரிந்து கொள்வது character இதுபோல் செய்ய முடியுமா என்ன?`



அருமையான கேள்வி.

நீ கேட்பது:

> `a + (-a) = 0` என்றால், `a` ஒரு character ஆக இருந்தால் இதுபோல் செய்ய முடியுமா?

பதில்:

**இல்லை. பொதுவாக முடியாது.**

ஏனெனில் `a + (-a) = 0` என்பது எல்லா பொருட்களுக்கும் பொருந்தும் விதி அல்ல. இது **Integers, Rational Numbers, Real Numbers, Complex Numbers** போன்ற குறிப்பிட்ட Mathematical Systems-ல் மட்டுமே பொருந்தும்.

---

### Number-ல்

உதாரணம்:

```text
5 + (-5) = 0

100 + (-100) = 0

-7 + 7 = 0
```

இங்கு `-a` என்பதற்கு அர்த்தம் இருக்கிறது.

---

### Character-ல்

உதாரணம்:

```text
'A'
'B'
'C'
```

இப்போது

```text
-A
```

என்றால் என்ன?

அதற்கு சாதாரண கணிதத்தில் அர்த்தமே இல்லை.

அதனால்

```text
'A' + (-'A')
```

என்பதும் அர்த்தமற்றது.

---

### கணிதத்தின் முக்கியமான Idea

`a + (-a) = 0` என்பது

> "ஒவ்வொரு உறுப்புக்கும் (element) அதை cancel செய்யும் ஒரு opposite element இருக்க வேண்டும்"

என்று சொல்கிறது.

உதாரணம்:

#### Temperature

```text
+10°
```

ஐ cancel செய்ய

```text
-10°
```

---

#### Bank

```text
+100 ரூபாய்
```

ஐ cancel செய்ய

```text
-100 ரூபாய்
```

---

### Characters-க்கு cancel element இல்லையா?

சாதாரண Characters-க்கு இல்லை.

```text
A
B
C
```

இவைகளுக்கு

```text
-A
-B
-C
```

என்ற mathematical opposite வரையறுக்கப்படவில்லை.

---

### ஆனால் சில Mathematical Objects-க்கு உண்டு

உதாரணம்:

Vector

```text
v = (3, 4)
```

இதன் opposite:

```text
-v = (-3, -4)
```

அப்போது

```text
v + (-v) = (0,0)
```

ஆகும்.

இதுவும் `a + (-a) = 0` என்ற அதே idea தான்.

---

### ஆழமான உண்மை

`a + (-a) = 0` என்பது Numbers பற்றிய விதி மட்டும் அல்ல.

ஒரு Mathematical System-ல்:

* Addition இருக்க வேண்டும்.
* Zero போன்ற identity element இருக்க வேண்டும்.
* ஒவ்வொரு element-க்கும் inverse இருக்க வேண்டும்.

அப்போதுதான்

```text
a + (-a) = 0
```

என்ற கருத்து பொருள் பெறும்.

Characters (`A`, `B`, `C`) இவற்றில் அப்படி ஒரு அமைப்பு (structure) இல்லாததால், பொதுவாக இதை செய்ய முடியாது.

அதனால் ஆசிரியர் `a` என்று எழுதும்போது,

> "எந்த character-ஐயும் குறிக்கவில்லை; inverse உடைய ஒரு எண்ணையோ அல்லது அதுபோன்ற mathematical object-ஐயோ குறிக்கிறார்."

என்று புரிந்துகொள்ள வேண்டும்.
