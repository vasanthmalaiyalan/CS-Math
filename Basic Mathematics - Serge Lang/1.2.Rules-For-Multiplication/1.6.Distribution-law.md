இல்லை. இங்கே `0b²` வராது. `0b` தான் வரும்.

நாம் அந்த line-ஐ மெதுவாகப் பார்ப்போம்:

```text
ab + (-a)b
```

Distributive law:

```text
= (a + (-a))b
```

இது சரிதானே?

ஏனெனில்

```text
xb + yb = (x+y)b
```

---

இப்போது:

```text
a + (-a)
```

என்ன?

Addition chapter-ல்:

```text
a + (-a) = 0
```

---

அதனால்:

```text
(a + (-a))b
=
0b
```

---

இங்கே கவனி:

```text
(a + (-a))
```

என்ற முழு bracket தான் `0` ஆனது.

அதனால்:

```text
0 × b
```

கிடைக்கிறது.

---

`0b²` வர வேண்டுமென்றால்:

```text
(a + (-a))bb
```

அல்லது

```text
((a+(-a))b)b
```

போன்ற ஒரு extra `b` இருக்க வேண்டும்.

ஆனால் நம்மிடம் இருப்பது:

```text
(a + (-a))b
```

ஒரே ஒரு `b` மட்டுமே.

---

ஒரு எண் உதாரணம்:

```text
a = 5
b = 3
```

அப்போது:

```text
ab + (-a)b
```

=

```text
5×3 + (-5)×3
```

=

```text
15 + (-15)
```

=

```text
0
```

---

புத்தகத்தின் வழி:

```text
(5 + (-5))×3
```

=

```text
0×3
```

=

```text
0
```

---

இங்கே எங்கும்:

```text
0×3²
```

வரவில்லை.

அதனால் சரியானது:

```text
(a + (-a))b
=
0b
=
0
```

✅

`0b²` ❌ அல்ல.

நீ `0b²` என்று நினைத்தது, `(a+(-a))b`-ஐ பார்க்கும்போது `ab + (-a)b`-ல் இருந்த இரண்டு `b`-களை மனதில் சேர்த்துவிட்டதால் இருக்கலாம். ஆனால் உண்மையில் இரு terms-லும் இருக்கும் **அதே ஒரு `b` factor**-ஐ வெளியே எடுத்திருக்கிறோம் (factoring), புதிய `b` ஒன்றை உருவாக்கவில்லை.
