இந்த Section (§4 Inequalities) மிகவும் எளிதாகத் தோன்றும். ஆனால் உண்மையில் Serge Lang இங்கே **"Greater than (`>`), Less than (`<`)" என்பவற்றை புதிதாக define செய்கிறார்.**

முக்கியமாக அவர் சொல்ல வருவது:

> **`>` என்பது ஒரு primitive symbol இல்லை. அதை positivity (`>0`) மூலம் define செய்யலாம்.**

இதை வரி வரியாகப் பார்ப்போம்.

---

# 1.

ஆசிரியர் முதலில் நினைவூட்டுகிறார்:

```text
a > 0
```

என்றால்

> **`a` என்பது Positive Number.**

இதில் புதிதாக எதுவும் இல்லை.

---

# 2.

இப்போது அவர் புதிய Definition கொடுக்கிறார்.

```text
a > b
```

என்பதற்கு அர்த்தம் என்ன?

அவர் சொல்கிறார்:

```text
a > b
```

என்பதற்குப் பதிலாக

```text
a - b > 0
```

என்று எழுதுவோம்.

இது ஒரு **Definition**.

---

## ஏன் இப்படிச் செய்கிறார்?

உதாரணம்:

```text
8 > 5
```

ஏன்?

ஏனெனில்

```text
8 - 5 = 3
```

---

```text
3 > 0
```

ஆகவே

```text
8 > 5
```

---

அதாவது

நாம் இரண்டு எண்களை ஒப்பிடுவதில்லை.

நாம்

**அவற்றின் வித்தியாசம் (difference)** positive-ஆக இருக்கிறதா என்று பார்க்கிறோம்.

---

## இன்னொரு உதாரணம்

```text
10 > 7
```

---

Difference

```text
10-7=3
```

---

3 positive.

அதனால்

```text
10>7
```

## Negative Number Example

புத்தகத்தில்:

```text

```

என்று சொல்கிறார்.

முதலில் பார்க்கும்போது இது வித்தியாசமாக இருக்கும்.

---

Definition பயன்படுத்து.

```text
-1-(-2)
```

---

Minus Minus

```text
-1+2
```

---

```text
1
```

---

1 positive.

அதனால்

```text
-1>-2
```

---

இது மிகவும் முக்கியமான idea.

**Negative numbers-ஐ compare செய்யும்போதும் அதே Definition.**

# இந்த Chapter-ன் Deep Philosophy

இதுவரை Serge Lang ஒவ்வொரு கருத்தையும் மிகவும் குறைந்த Definition-களிலிருந்து உருவாக்கினார்.

* `a>b`-ஐ primitive-ஆக எடுத்துக்கொள்ளவில்லை.
* அதை

```text
a-b>0
```

என்ற **positivity**-யிலிருந்து define செய்தார்.

அதாவது, அவரது பார்வையில்:

> **"Greater than" என்பது தனி concept இல்லை. "Difference positive-ஆக இருப்பது" என்பதையே வேறு notation-ல் எழுதுகிறோம்.**

அதனால்தான் பின்னர் வரும் அனைத்து inequality rules (`>`, `<`, `≥`) கூட **positivity** மற்றும் algebra rules (addition, multiplication) மட்டுமே பயன்படுத்தி நிரூபிக்க முடியும்.

இது Serge Lang-ன் பாணி: **புதிய symbols-ஐ அறிமுகப்படுத்தும்போதும், முடிந்தவரை பழைய அடிப்படை கருத்துகளிலிருந்து அவற்றை உருவாக்குவது.**

------------------------------

``3.
அடுத்தது
a<0
என்பதை
-a>0
என்று எழுதுவோம்.
இதன் அர்த்தம்:
ஒரு எண் Negative என்றால்
அதன் Additive Inverse Positive.`நீ சொல்லவரும் மீது சுத்தமா புரியவில்லை?`


பரவாயில்லை. இதை Serge Lang சொல்லும் விதத்தில் மிகவும் மெதுவாகப் பார்ப்போம்.

முதலில் இந்த வரியை மட்டும் பார்ப்போம்:

```text
a < 0
```

இதன் அர்த்தம் என்ன?

> **`a` என்பது ஒரு negative number.**

உதாரணம்:

```text
a = -5
```

அப்போது:

```text
a < 0
```

உண்மை.

---

இப்போது ஆசிரியர் சொல்கிறார்:

```text
a < 0
```

என்று எழுதுவதற்குப் பதிலாக

```text
-a > 0
```

என்றும் எழுதலாம்.

ஏன்?

நம்முடைய உதாரணத்தைப் பயன்படுத்துவோம்.

```text
a = -5
```

அப்போது

```text
-a
```

என்பது

```text
-(-5)
```

---

நாம் ஏற்கனவே கற்றுக்கொண்டோம்:

```text
-(-5) = 5
```

ஆக:

```text
-a = 5
```

இப்போது:

```text
5 > 0
```

உண்மை.

அதனால்:

```text
a < 0
```

மற்றும்

```text
-a > 0
```

இரண்டும் ஒரே விஷயத்தைச் சொல்கின்றன.

---

## இன்னொரு உதாரணம்

வைத்துக்கொள்வோம்:

```text
a = -12
```

அப்போது:

```text
a < 0
```

உண்மை.

இப்போது:

```text
-a
=
-(-12)
=
12
```

மேலும்:

```text
12 > 0
```

இதுவும் உண்மை.

---

## Positive Number எடுத்தால்?

```text
a = 7
```

அப்போது:

```text
a < 0
```

பொய்.

இப்போது:

```text
-a = -7
```

அதாவது:

```text
-7 > 0
```

இதுவும் பொய்.

இரண்டும் மீண்டும் ஒன்றாகவே இருக்கின்றன.

---

## ஆசிரியர் ஏன் இப்படிச் சொல்கிறார்?

அவர் புதிய definition கொடுக்கவில்லை.

அவர் சொல்ல வருவது:

> **ஒரு எண்ணை negative என்று சொல்வது, அதன் additive inverse positive என்று சொல்வதற்கு சமம்.**

உதாரணம்:

| `a`   | `a < 0` | `-a` | `-a > 0` |
| ----- | ------- | ---- | -------- |
| `-5`  | ✅       | `5`  | ✅        |
| `-12` | ✅       | `12` | ✅        |
| `7`   | ❌       | `-7` | ❌        |

---

### முக்கியமான விஷயம்

நீ முன்பு additive inverse-ஐ நன்றாகப் புரிந்துகொண்டாய்.

* `5`-ன் additive inverse → `-5`
* `-5`-ன் additive inverse → `5`

அதைத்தான் Serge Lang இங்கே பயன்படுத்துகிறார்.

அதனால்,

```text
a < 0
```

என்றால்

"`a` negative" என்று சொல்லலாம்.

அதே விஷயத்தை

```text
-a > 0
```

என்றும் சொல்லலாம்.

அதாவது:

> **"`a`-வின் additive inverse positive."**

---

இப்போது நான் ஒரு சிறிய கேள்வி கேட்கிறேன்.

```text
a = -8
```

என்றால்,

1. `a < 0` உண்மையா?
2. `-a` என்ன?
3. `-a > 0` உண்மையா?

இந்த மூன்றுக்கும் நீயே பதில் சொல்லிப் பார். அதிலிருந்து இந்த concept முழுவதும் தெளிவாகிவிடும்.
