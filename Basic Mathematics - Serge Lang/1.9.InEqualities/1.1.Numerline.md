# IN1

Rule:

```text
a>b
b>c
--------
a>c
```

Number Line-ல் பார்ப்போம்.

```text
<------------------------------>

0    2       7          10
     c       b           a
```

இங்கே:

* `10` என்பது `7`-க்கு வலது பக்கம்.
* `7` என்பது `2`-க்கு வலது பக்கம்.

அப்படியானால்,

**10 கண்டிப்பாக 2-க்கும் வலது பக்கம் தான்.**

அதனால்:

```text
10>2
```

இங்கே formula இல்லை.

**Number Line-ஐ பார்த்தாலே தெரிகிறது.**

# IN2

Rule:

```text
a>b
```

மேலும்

```text
c>0
```

அதாவது `c` positive.

உதாரணம்:

```text
8>5
```

Number Line:

```text
0      5        8
       b        a
```

இப்போது இரண்டையும் 3-ஆல் multiply செய்.

```text
8×3 = 24
5×3 = 15
```

Number Line:

```text
0          15                 24
           bc                 ac
```

இங்கே என்ன நடந்தது?

5 இருந்த இடம் 15 ஆனது.

8 இருந்த இடம் 24 ஆனது.

**இரண்டும் வலது பக்கம் தள்ளப்பட்டன.**

ஆனால் வரிசை மாறவில்லையே!

முன்பு

```text
8 > 5
```

இப்போது

```text
24 > 15
```

---

## ஏன்?

Positive number-ஆல் multiply செய்தால்,

அனைத்து distances-ம் ஒரே ratio-வில் பெரிதாகும்.

உதாரணம்:

```text
5 -------- 8
```

Distance:

```text
3
```

```text

```

Distance:

```text

```

Distance பெரிதானது.

ஆனால் order மாறவில்லை.

# IN3

இப்போதுதான் வித்தியாசம்.

```text
8>5
```

இப்போது

```text
-2
```

ஆல் multiply செய்.

```text
8×(-2)=-16
```

```text
5×(-2)=-10
```

Number Line:

```text
-16      -10        0
```

இங்கே

```text
-16 < -10
```

ஏன்?

ஏனெனில் negative-ஆல் multiply செய்தால் Number Line **mirror (reflection)** ஆகிறது.

முன்பு:

```text
5 -------- 8
```

இப்போது:

```text
-16 -------- -10
```

வலது-இடது மாறிவிட்டது.

அதனால்தான்:

```text
8>5
```

↓

```text
-16<-10
```
