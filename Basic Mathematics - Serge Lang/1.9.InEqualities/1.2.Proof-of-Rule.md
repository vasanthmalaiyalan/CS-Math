# பகுதி 1 - Rule IN2-ன் அர்த்தம்

முதலில் அவர் சொல்கிறார்:

> **Rule IN2 expresses the fact that an inequality which is multiplied by a positive number is preserved.**

அதாவது,

```text
8 > 5
```

இருபக்கமும் `3`-ஆல் பெருக்கினால்

```text
24 > 15
```

`>` மாறவில்லை.

---

பிறகு அவர் சொல்கிறார்:

> **Rule IN3 tells us that if we multiply both sides by a negative number, then the inequality gets reversed.**

உதாரணம்:

```text
1 < 3
```

இப்போது `-2`-ஆல் பெருக்கு.

```text
1×(-2) = -2
3×(-2) = -6
```

ஆக

```text
-2 > -6
```

கவனித்தாயா?

முன்பு

```text
1 < 3
```

இப்போது

```text
-2 > -6
```

`<` என்பது `>` ஆக மாறிவிட்டது.

---

## ஏன்?

Number Line-ல் பார்ப்போம்.

முதலில்

```text
0-----1----------3----->
```

1 என்பது 3-க்கு இடது பக்கம்.

---

இப்போது எல்லாவற்றையும் `-2`-ஆல் multiply செய்.

```text
1  → -2
3  → -6
```

Number Line:

```text
<----- -6 ----- -2 -----0
```

இப்போது `-2` வலது பக்கம்.

அதனால்

```text
-2 > -6
```

Negative multiply என்பது **Number Line-ஐ mirror (flip)** செய்கிறது.

# இந்த இரண்டு பக்கங்களின் மிக முக்கியமான Philosophy

Serge Lang உனக்கு ஒரு **Formula** கற்றுக்கொடுக்கவில்லை.

அவர் ஒரு **Decision Process** கற்றுக்கொடுக்கிறார்.

ஒரு inequality-ல் denominator அல்லது multiplier வந்தால், முதலில் கேள்:

> **"இது Positive-ஆ? Negative-ஆ?"**

அதன்பிறகுதான் அடுத்த step.

அதனால் அவர் எப்போதும்:

1. முதலில் sign (`>0` அல்லது `<0`) கண்டுபிடிக்கிறார்.
2. அதன் பிறகுதான் multiply அல்லது divide செய்கிறார்.

இந்த பழக்கம் மிகவும் முக்கியம். உயர்நிலை Algebra, Calculus, Analysis எல்லாவற்றிலும் இதே சிந்தனைதான் தொடர்ந்து பயன்படுகிறது.

--------------------------------------

`சரி இப்படி இருந்தது `(3x+5)/(x-4)<2`எப்படி இப்படியானது `3x+5<2(x-4)`?`

இதுதான் இந்த example-ல் **மிக முக்கியமான step**. இதை புரிந்துகொண்டால் inequality solving-ல் பாதி முடிந்தது.

நமக்கு கொடுக்கப்பட்டிருப்பது:

```text
(3x+5)/(x-4) < 2
```

முதலில் ஆசிரியர் நேரடியாக multiply செய்யவில்லை.

அவர் முதலில் கேட்கிறார்:

> **`x-4` Positive-ஆ? Negative-ஆ?**

ஏனெனில் அதன் sign தெரியாமல் multiply செய்யக் கூடாது.

---

## Case 1

வைத்துக்கொள்வோம்:

```text
x > 4
```

அப்போது:

```text
x-4 > 0
```

அதாவது denominator **Positive**.

இப்போது Positive number-ஆல் இருபக்கமும் multiply செய்யலாம்.

இடது பக்கம்:

```text
((3x+5)/(x-4)) × (x-4)
```

Fraction-ல் numerator, denominator cancel ஆகிவிடும்.

```text
= 3x+5
```

---

வலது பக்கம்:

```text
2 × (x-4)
```

அதுதான்

```text
2(x-4)
```

---

அதனால்

```text
(3x+5)/(x-4) < 2
```

மாறி

```text
3x+5 < 2(x-4)
```

ஆகிறது.

---

## ஏன் `<` மாறவில்லை?

ஏனெனில்

```text
x-4 > 0
```

Positive number-ஆல் multiply செய்தோம்.

IN2 சொல்வது:

```text
a>b
```

Positive-ஆல் multiply செய்தால்

**inequality மாறாது.**

---

## ஒரு எளிய Number Example

இதையே numbers-ல் பார்ப்போம்.

```text
3/2 < 2
```

உண்மையா?

ஆம்.

இப்போது denominator `2` Positive.

இருபக்கமும் `2`-ஆல் multiply செய்.

இடது:

```text
(3/2)×2 = 3
```

வலது:

```text
2×2 = 4
```

ஆக

```text
3<4
```

அதேதான் நடந்தது.

---

## இப்போது Case 2

வைத்துக்கொள்வோம்:

```text
x<4
```

அப்போது

```text
x-4<0
```

Negative.

இப்போது இருபக்கமும் `(x-4)`-ஆல் multiply செய்தால்,

IN3 படி inequality reverse ஆக வேண்டும்.

அதனால்

```text
(3x+5)/(x-4) < 2
```

மாறி

```text
3x+5 > 2(x-4)
```

ஆகிறது.

**கவனித்தாயா?**

* `x-4 > 0` என்றால்

```text
3x+5 < 2(x-4)
```

* `x-4 < 0` என்றால்

```text
3x+5 > 2(x-4)
```

**ஒரே equation. ஆனால் denominator-ன் sign மாறியதால் inequality direction மட்டும் மாறுகிறது.**

---

### Serge Lang ஏன் முதலில் `x>4` மற்றும் `x<4` என்று case பிரிக்கிறார்?

ஏனெனில் அவர் **ஒரு விதியை ஒருபோதும் உடைக்க மாட்டார்**.

அவருக்கு `(x-4)` positive-ஆ, negative-ஆ என்று தெரியாமல் இருந்தால்,

```text
(3x+5)/(x-4) < 2
```

இருந்து

```text
3x+5 < 2(x-4)
```

என்று நேரடியாக எழுத முடியாது.

அது கணித ரீதியாக தவறு.

**முதலில் sign தெரிந்த பிறகுதான் multiply செய்ய வேண்டும்** என்பதைத்தான் அவர் மிகவும் ஒழுக்கமாகக் கடைப்பிடிக்கிறார்.
