# 1. Place-value arithmetic-ன் மிகப் பெரிய சக்தி

முன்பு நாம் பார்த்தது:

```text
56×144

=
(50+6)(100+40+4)

=
5000+2000+200+600+240+24

=
8064
```

இதில் நாம் உண்மையில் rocks, rectangles, muffins, eggs எதையும் பயன்படுத்தவில்லை.

வெறும் symbols:

```text
5
6
1
4
4
```

இவற்றை விதிப்படி நகர்த்தினோம்.

அதாவது

> Place-value arithmetic என்பது quantity-களை symbolic manipulation-ஆக மாற்றிவிடுகிறது.

# 2. புரியாமலேயே arithmetic செய்ய முடியுமா?

முடியும்.

ஒரு குழந்தையை எடுத்துக்கொள்ளுங்கள்.

அவனுக்கு

```text
7×8=56
```

என்று மனப்பாடம்.

ஆனால்

"ஏன்?"

தெரியாது.

Long multiplication algorithm தெரியும்.

ஆனால் rectangle தெரியாது.

Distributive property தெரியாது.

Copies என்ற கருத்து தெரியாது.

அவன் வெறும்

```text
Step 1
Step 2
Carry
Step 3
```

என்று follow செய்கிறான்.

Lockhart இதை கிண்டலாக சொல்கிறார்:

> இப்படிப்பட்ட mechanical வேலைக்காக குழந்தைகளை training செய்து office worker ஆக்கலாம்.

# 4. Machine என்றால் என்ன?

Machine-க்கு

understanding தேவையில்லை.

அது செய்ய வேண்டியது:

```text

இந்த action செய்ய வேண்டும்.
```

அவ்வளவு தான்.

# 5. முக்கியமான principle

Machine ஒன்றை உருவாக்க

முதலில்

மனிதன் செய்யும் வேலையை

புரிதல் இல்லாத

சிறு சிறு steps-ஆக உடைக்க வேண்டும்.

உதாரணம்

நாம் multiplication செய்யும்போது:

```text
173×254
```

நமக்கு rectangle, area, quantity எல்லாம் தெரியும்.

ஆனால் machine-க்கு தெரியாது.

அதற்கு சொல்ல வேண்டியது:

```text
4×3

4×7

4×1

5×3

...

carry
add
```

அவ்வளவு தான்.

# 10. Meaningless physical interpretation

இதுதான் genius.

```text

```

```text
wheel ஒரு notch திரும்ப வேண்டும்.
```

அவ்வளவு.

Quantity இல்லை.

Meaning இல்லை.

வெறும்

motion.
