இந்த பகுதி முழுவதும் Paul Lockhart-ன் மிக அழகான தத்துவம். அவர் வெறும் calculator வரலாறு சொல்லவில்லை; **"மனிதன் செய்யும் வேலை → algorithm → machine → electronics → software"** என்ற பரிணாமத்தை காட்டுகிறார்.

# 3. Carry pins arithmetic புரிந்துகொள்ளாது

Counter-க்கு

```text
458 + 39
```

என்பது தெரியாது.

அதற்கு தெரியுவது:

```text
wheel rotate
carry pin engage
next wheel move
```

அவ்வளவுதான்.

ஆனால் விளைவு:

```text
458
+39
----
497
```

சரியாக வருகிறது.

அதனால்

**Understanding இல்லாமல் computation செய்ய முடியும்.**

# 10. Transistor revolution

1950களில் transistor வந்தது.

இதுவே electronics உலகின் carry pin.

Transistor:

3 terminals

```text
A
B
C
```

C-ல் voltage இருந்தால்

A ↔ B current flow.

இல்லையெனில்

flow இல்லை.

அதாவது

```text
IF condition
THEN react
```

இது carry pin செய்த வேலையே.
