இந்தப் பகுதி முழுவதும் Paul Lockhart-ன் மிகவும் ஆழமான பார்வை. அவர் "2+3=5" என்று கணக்குப் போடுவது பற்றி பேசவில்லை. மாறாக,

> **"எண்கள் என்றால் என்ன?", "கூட்டல் என்றால் என்ன?", "கழித்தல் உண்மையில் தேவையா?"**

என்ற கேள்விகளைப் பற்றி பேசுகிறார்.

மெதுவாகப் போவோம்.

---

# 1. Zero என்பது "ஒன்றுமில்லை" அல்ல

நாம் பள்ளியில்

```text
0 = nothing
```

என்று கற்றுக்கொள்கிறோம்.

ஆனால் modern algebra பார்வையில்,

0 என்பது

> "எந்த எண்ணுடன் கூட்டினாலும் அதை மாற்றாமல் விட்டுவிடும் entity"

அதாவது

```text
a + 0 = a
```

எந்த a-க்கும்.

உதாரணம்

```text
5+0=5

100+0=100

2/3+0=2/3
```

இதுதான் zero-வின் உண்மையான அடையாளம்.

Zero என்பது

"ஒன்றுமில்லை" என்பதல்ல.

மாறாக

**addition-இல் எதையும் செய்யாமல் இருப்பவன்.**

---

# 2. One என்றால்?

அதேபோல

1 என்பது

> எந்த எண்ணுடன் பெருக்கினாலும் அதை மாற்றாமல் விடுவது.

```text
a×1=a
```

உதாரணம்

```text
5×1=5

100×1=100

2/3×1=2/3
```

எனவே

1 என்பது

"ஒரு பொருள்"

என்பதல்ல.

Multiplication-இல்

"எதுவும் செய்யாதவன்".

---

# 3. அப்படியானால் Addition என்றால் என்ன?

சாதாரணமாக

கற்கள் இரண்டு குவியலை சேர்ப்பது.

ஆனால் mathematician இப்படி யோசிக்கிறார்:

Addition என்றால்

"சேர்த்தல்" என்பதே முக்கியமில்லை.

அது எந்த pattern-ஐ பின்பற்றுகிறது என்பதே முக்கியம்.

# 4. Hamster analogy

Lockhart சொல்கிறார்:

எண்களை உயிரினங்களாக நினைத்துக் கொள்ளுங்கள்.

ஒவ்வொரு ஜோடி hamsters-க்கும்

ஒரு hamster-ஐ assign செய்கிறோம்.

உதாரணம்

```text
(2,3) → 5

(7,4) → 11
```

இதுதான் addition.

அதாவது

Addition என்பது

"process"

இல்லை.

ஒரு

**assignment rule**

மட்டும்.

---

# 5. Mathematical operation = Assignment

இரண்டு எண்களை கொடுத்தால்

ஒரு எண் திரும்ப வருகிறது.

அதுவே operation.

உதாரணம்

```text
2+3 → 5

2×3 → 6
```

அதனால்

கூட்டுவது என்பது கற்களை சேர்ப்பது அல்ல.

வெறும்

ஒரு pair-க்கு ஒரு output assign செய்வது.

# 6. Addition-ன் மிக அழகான property

### Commutative property

```text
a+b=b+a
```

உதாரணம்

```text
5+3=3+5
```

order முக்கியமில்லை.

இது மிகவும் special property.

Division-க்கு இது இல்லை.

ஏனெனில்

```text
5÷3 ≠ 3÷5
```

---

இதையே

Lockhart universal property என்கிறார்.

எல்லா எண்களுக்கும் இது உண்மை.

---

# 7. Associative property

மூன்று எண்கள் இருந்தால்

```text
(2+3)+7
```

அல்லது

```text
2+(3+7)
```

இரண்டும்

```text
12
```

தான்.

அதாவது

```text
(a+b)+c
=
a+(b+c)
```

grouping முக்கியமில்லை.

இதுதான் associative property.

---

அதனால்

```text
2+3+7
```

என்று brackets இல்லாமல் எழுதலாம்.

# 8. எல்லா operations-க்கும் இது கிடையாது

Subtraction:

```text

=3

=7
```


வேறு answer.

அதனால் division associative இல்லை.

> **You do not understand a mathematical object by asking "What is it?"**
