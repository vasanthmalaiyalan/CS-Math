இது Paul Lockhart-ன் மிக அழகான பகுதிகளில் ஒன்று. இங்கே அவர் **negative numbers ஏன் உருவாக்கப்பட்டன?**, **(-2)×(-3)=6 ஏன்?** என்பதைக் "rule" ஆக அல்ல, "அழகிய கட்டமைப்பு (beautiful structure)" ஆக விளக்குகிறார்.

---

# 1. கழித்தல் (Subtraction) அழகாக நடந்து கொள்ளாது

கூட்டல்:

```text
3+5 = 8
5+3 = 8
```

Order மாற்றினாலும் பிரச்சனை இல்லை.

ஆனால்

```text
8-5 = 3
5-8 = ?
```

இங்கே சிக்கல் வருகிறது.

---

மேலும்

```text
3-5
```

என்றால்?

Piles of rocks பார்வையில்:

3 கற்கள் இருக்கின்றன.

5 கற்களை எடுக்க சொல்கிறாய்.

அது முடியாது.

அதனால் subtraction-க்கு ஒரு restriction உள்ளது.

> எடுத்துக் கொள்ளும் எண்ணை விட குறைவாக இருக்க வேண்டும்.

இதுதான் Lockhart-க்கு பிடிக்கவில்லை.

---

# 2. Addition-ல் இந்த restriction இல்லை

எந்த இரண்டு எண்களும்

```text
a+b
```

என்றால் answer இருக்கிறது.

ஆனால் subtraction-ல்

```text
a-b
```

என்றால்

a≥b என்றால் மட்டுமே answer.

இது symmetry இல்லை.

# 3. Mathematician என்ன சொல்கிறார்?

Practical மனிதன்:

> "அதுதான் உலகம்."

Mathematician:

> "இல்லை. நான் புதிய எண்களை உருவாக்குகிறேன்."

Fractions எப்படி remainder-ஐ உடைத்து வந்ததோ

அதே போல

negative numbers உருவாக்கப்படுகின்றன.

# 4. Extension problem

இது modern mathematics-ல் மிகவும் முக்கியமான idea.

ஒரு system இருக்கிறது.

ஆனால் அது முழுமையற்றதாக இருக்கிறது.

அதை விரிவுபடுத்தி (extend) புதிய entities சேர்க்கிறோம்.

அதாவது

Natural numbers

```text
1,2,3,...
```

↓

Integers

```text
..., -3,-2,-1,0,1,2,3,...
```

↓

Fractions

```text
2/3
```

↓

Real numbers

↓

Complex numbers

எல்லாமே extension problems.

# 5. Anti-sheep idea

2 sheep இருக்கிறது.

அதை அழிக்க வேண்டும்.

அதற்காக

2 anti-sheep உருவாக்குகிறோம்.

அப்போது

```text
2 sheep + 2 anti-sheep = 0
```

இது metaphor மட்டுமே.

---

இதையே abstract-ஆக

"-2"

என்று அழைக்கிறோம்.

Definition:

> -2 என்பது 2 உடன் கூட்டினால் 0 வரும் entity.

அதாவது

```text
2+(-2)=0
```

---

# 6. Negative sign என்பது label அல்ல

நாம்

```text
-2
```

என்று பார்க்கும்போது

அது ஒரு பெயர் அல்ல.

ஒரு operation.

Negation operator.

2-ஐ opposite ஆக்குகிறது.

---

அதனால்

```text
-(-2)=2
```

ஏன்?

ஏனெனில்

negative-ஐ மீண்டும் negative செய்தால்

மீண்டும் original கிடைக்கும்.

Coin-ஐ இருமுறை flip செய்வது போல.

---

# 7. -0 என்றால்?

0-வின் opposite என்ன?

0 தான்.

ஏனெனில்

```text
0+0=0
```

எனவே

```text
-0=0
```

# 8. Subtraction தேவையில்லாத operation

Lockhart சொல்வது:

```text
8-5
```

என்பது

உண்மையில்

```text
8+(-5)
```

தான்.

அதனால் subtraction என்ற தனி operation தேவையில்லை.

கூட்டல் மட்டும் போதும்.

Negative numbers இருந்தால்

கழித்தலை addition-ஆக மாற்றிவிடலாம்.

---

### 5−3

```text
5+(-3)=2
```

---

### 3−5

```text
3+(-5)=-2
```

அழகான symmetry.

# 9. கழித்தல் negative எண்ணை கழித்தால்?

உதாரணம்

```text
7-(-2)
```

என்பது

```text
7+(+2)
```

ஏன்?

ஏனெனில்

negative number-ஐ remove செய்வது

positive number சேர்ப்பதற்கு சமம்.

---

Sheep metaphor:

2 anti-sheep இருக்கின்றன.

அவற்றை நீ remove செய்கிறாய்.

அதாவது

negative ஐ remove செய்கிறாய்.

அதன் விளைவு:

2 sheep அதிகரிக்கும்.

அதனால்

```text
7-(-2)=9
```

# 11. Multiplication with negatives

இதுவரை

negative numbers-க்கு addition behavior மட்டுமே கொடுத்தோம்.

ஆனால்

```text
(-2)×(-3)
```

என்றால்?

"minus two copies of minus three"

என்பது physical meaning இல்லாதது.

---

அப்போது

Lockhart சொல்கிறார்:

Meaning பற்றி கவலைப்படாதீர்கள்.

Patterns-ஐ காப்பாற்றுங்கள்.

---

# 12. Symmetry காப்பாற்ற வேண்டும்

நமக்கு

```text
2×(-1)=-2
```

தெரியும்.

Multiplication commutative ஆக இருக்க வேண்டும்.

அதனால்

```text
(-1)×2=-2
```

ஆக வேண்டும்.

---

இதிலிருந்து

"-1 உடன் multiply செய்வது"

என்பது

number-ஐ negate செய்வது என்று புரிகிறது.

```text
(-1)×5=-5
```

```text
(-1)×(2/3)=-(2/3)
```

---

# 13. Copy machine metaphor

Positive multiplication:

scale only.

Negative multiplication:

scale + flip.

---

உதாரணம்

```text
×3
```

மூன்று மடங்கு பெரிது.

```text
×(-3)
```

மூன்று மடங்கு பெரிது + zero-வின் மறுபக்கம் flip.

---

# 14. ஏன் (-2)×(-3)=+6 ?

முதலில்

```text
-3
```

உருவாக்கு.

அதாவது

3 மடங்கு enlarge செய்து

ஒருமுறை flip.

பிறகு

```text
×(-2)
```

செய்.

2 மடங்கு enlarge செய்.

மீண்டும் flip செய்.

---

மொத்தம்:

Scale:

```text
3×2=6
```

Flip:

இரண்டு முறை.

```text
flip × flip = no flip
```

எனவே

```text
(-2)×(-3)=6
```

---

இதுதான் school rule அல்ல.

இது symmetry-யை காப்பாற்றியதால் forced answer.

வேறு answer கொடுத்தால்

multiplication structure உடைந்து போய்விடும்.

---

# 15. Reciprocal

```text
2 × 1/2 =1
```

அதனால்

1/2 என்பது reciprocal.

அதேபோல்

```text
(-2)×(-1/2)=1
```

அதனால்

```text
-1/2
```

என்பது

-2-ன் reciprocal.

# Lockhart-ன் மிக முக்கியமான கருத்து

Mathematics-ல்

நாம் முதலில்

meaning உருவாக்க மாட்டோம்.

முதலில் patterns-ஐ காப்பாற்றுவோம்.

அந்த patterns-ஐ காப்பாற்றும்போது

புதிய entities தானாக பிறக்கின்றன.

Fractions பிறந்தது.

Negative numbers பிறந்தது.

Complex numbers பிறந்தது.

எல்லாவற்றிற்கும் பின்னால் இருக்கும் philosophy:

> **"What value should this object have so that all the beautiful patterns continue to hold?"**

அதனால்தான்

```text
(-2)×(-3)=6
```

என்பது ஒரு rule அல்ல.

அது கணிதத்தின் symmetry மற்றும் elegance-ன் தவிர்க்க முடியாத விளைவு.
