அருமையான பகுதி இது. Paul Lockhart இங்கே உண்மையில் **fraction arithmetic** கற்றுக் கொடுக்கவில்லை; "fraction என்றால் என்ன?" என்ற ஆழமான கருத்தை உருவாக்குகிறார். ஒவ்வொரு கருத்தையும் ஆழமாகப் பார்ப்போம்.

---

# 1. Fraction என்பது புதிய வகை எண்

முன்பு நமக்கு

```
1,2,3,4,5,...
```

என்ற counting numbers மட்டும் இருந்தன.

இப்போது division காரணமாக

```
1/2
2/3
57/11
```

போன்ற புதிய எண்கள் வந்துவிட்டன.

உதாரணம்:

```
57/11
```

என்றால்,

* ஒன்றை 11 சம பாகங்களாக வெட்டு
* அந்தப் பாகங்களில் 57 எடுத்துக்கொள்

அவ்வளவுதான்.

---

# 2. ஒரே denominator இருந்தால் comparison மிகவும் எளிது

உதாரணம்:

```
9/11
7/11
```

இரண்டுமே elevenths.

அதாவது இரண்டிலும் piece size ஒன்றே.

அதனால்:

9 pieces > 7 pieces

எனவே

```
9/11 > 7/11
```

---

# 3. ஒரே numerator இருந்தாலும் comparison எளிது

உதாரணம்:

```
4/5
4/7
```

இரண்டிலும் 4 pieces இருக்கிறது.

ஆனால்:

ஒரு fifth > ஒரு seventh

ஏனெனில்

ஒரு பொருளை 5 பாகமாக வெட்டினால் கிடைக்கும் துண்டு,

7 பாகமாக வெட்டினால் கிடைக்கும் துண்டை விட பெரியது.

எனவே

```
4/5 > 4/7
```

---

# 4. ஆனால்

```
3/7
2/5
```

?

இங்கு பிரச்சினை:

* ஒன்று sevenths எண்ணுகிறது.
* மற்றொன்று fifths எண்ணுகிறது.

இது

> 3 apples vs 2 oranges

மாதிரி.

நேரடியாக compare செய்ய முடியாது.

# 5. Common language வேண்டும்

உதாரணம்:

ஒருவன் dozen-ஆக எண்ணுகிறான்.

```
2 dozen
```

மற்றவன் egg-ஆக எண்ணுகிறான்.

```
24 eggs
```

இருவரும் ஒரே quantity பற்றி பேசுகிறார்கள்.

ஆனால் language வேறு.

அதனால் எல்லாவற்றையும் egg ஆக மாற்றினால் comparison எளிது.

Fraction-ல அதைத்தான் common denominator என்பார்கள்.

Lockhart முழுப் பகுதியில் சொல்லும் மையக் கருத்து:

> Fraction arithmetic என்பது புதிய விதி அல்ல.
>
> அது "எந்த unit-ல் எண்ணுகிறோம்?" என்ற language பிரச்சினை மட்டுமே.
>
> Common denominator என்பது language translation.
>
> Addition என்பது counting.
>
> Fraction என்பது ஒரு object அல்ல; அது ஒரு quantity-யை விவரிக்கும் பல பெயர்களில் ஒன்று.

இதிலிருந்து அடுத்த மிக அழகான பகுதி ஆரம்பிக்கிறது:

**Fraction multiplication மற்றும் division ஏன் numerator × numerator, denominator × denominator ஆகிறது?**
அதை Lockhart மிகவும் ஆழமாக "copy machine" மற்றும் "scaling" பார்வையில் விளக்குகிறார்.


-------------------------------------------------

``3.25
=
3 whole + 2 tenths + 5 hundredths
=
3 + 2/10 + 5/100
=
300/100 + 20/100 + 5/100
=
325/100
=
13/4`இதில் எதற்கு திடீர்னு இது எப்படி `3 + 2/10 + 5/100`இப்படி மாறியது `300/100 + 20/100 + 5/100``


இதுதான் decimal notation-ன் உண்மையான அர்த்தம். மனப்பாடம் செய்யாமல் புரிந்துகொண்டால் இது மிகவும் இயல்பாகத் தோன்றும்.

நாம்

```text
3.25
```

என்று எழுதும்போது, point-க்கு இடதுபுறம் whole numbers, வலதுபுறம் fractional place values.

அதாவது

```text
3.25
=
3 wholes + 2 tenths + 5 hundredths
```

ஏன்?

ஏனெனில் decimal system-ல்

```text
0.1 = 1/10
0.01 = 1/100
0.001 = 1/1000
```

அதனால்

```text
0.25
```

என்பது

```text
0.2 + 0.05
```

ஆகும்.

இப்போது,

```text
0.2 = 2 × 0.1 = 2 × (1/10) = 2/10
```

மேலும்,

```text
0.05 = 5 × 0.01 = 5 × (1/100) = 5/100
```

எனவே

```text
3.25
=
3 + 2/10 + 5/100
```

இங்கே திடீரென்று எதுவும் நடக்கவில்லை. decimal notation-ஐ fraction notation-ஆக மொழிபெயர்க்கிறோம்.

---

இப்போது ஏன்

```text
3 + 2/10 + 5/100
```

இருந்து

```text
300/100 + 20/100 + 5/100
```

என்று மாற்றினோம்?

ஏனெனில் எல்லாவற்றையும் ஒரே unit-ல் (hundredths) அளக்க விரும்புகிறோம்.

### முதலில்

3 wholes

ஒரு whole = 100/100

அதனால்

```text
3
=
3 × 100/100
=
300/100
```

---

அடுத்து

```text
2/10
```

ஒரு tenth = 10 hundredths

அதனால்

```text
2/10
=
20/100
```

---

இப்போது எல்லாமே hundredths ஆகிவிட்டது:

```text
3.25
=
300/100 + 20/100 + 5/100
```

இப்போது pieces எல்லாம் ஒரே அளவு (hundredths).

அதனால் numerator-களை மட்டும் கூட்டலாம்:

```text
=
(300+20+5)/100
=
325/100
```

பிறகு 25-ஆல் மேலேயும் கீழேயும் வகுத்தால்:

```text
325/100
=
13/4
```

### Lockhart பார்வையில்

இதன் அர்த்தம்:

```text
3.25
```

என்பது

**3 wholes + 2 tenths + 5 hundredths**

என்றும் பார்க்கலாம்,

அல்லது

**325 hundredths**

என்றும் பார்க்கலாம்.

இரண்டும் ஒரே quantity.

எப்படி

```text
1 dozen = 12 eggs
```

என்று சொல்வோமோ,

அதேபோல்

```text
3.25 = 325 hundredths = 13 quarters = 3¼
```

இவை எல்லாம் ஒரே quantity-க்கு வெவ்வேறு பெயர்கள் மட்டுமே.

இதுதான் Paul Lockhart அடிக்கடி கூறும் மிகப் பெரிய கருத்து:

> Arithmetic என்பது எண்களை மாற்றுவது அல்ல.
>
> அதே quantity-க்கு புதிய பெயர் (representation) கொடுப்பது.

---------------------------------------------


``ஏனெனில் எல்லாவற்றையும் ஒரே unit-ல் (hundredths) அளக்க விரும்புகிறோம்.
முதலில்
3 wholes
ஒரு whole = 100/100
அதனால்
Plain text
3
=
3 × 100/100
=
300/100`ஏன் நூறால் அளக்க விரும்ப வேண்டும்`


மிக அருமையான கேள்வி. **நூறால் (100) அளக்க வேண்டும் என்பது கட்டாயம் இல்லை.** அதுதான் முக்கியமான விஷயம்.

நாம்

```text
3 + 2/10 + 5/100
```

என்ற இடத்தில்

```text
3 = 300/100
2/10 = 20/100
```

என்று மாற்றியது, **எல்லாவற்றையும் hundredths என்ற ஒரே unit-ல் எழுதுவதற்காக மட்டுமே.**

### ஏன் ஒரே unit வேண்டும்?

ஏனெனில்

```text
3 wholes + 2 tenths + 5 hundredths
```

என்பதில் மூன்று விதமான units இருக்கின்றன:

* whole
* tenth
* hundredth

இவை apples, oranges, bananas மாதிரி வெவ்வேறு units.

அவற்றை நேராக சேர்க்க முடியாது.

அதனால் எல்லாவற்றையும் ஒரே unit-ஆக மாற்றுகிறோம்.

---

### ஏன் hundredths?

ஏனெனில் ஏற்கனவே

```text
5/100
```

என்று hundredths உள்ளது.

அதனால்:

```text
3 = 300/100
2/10 = 20/100
5/100 = 5/100
```

இப்போது எல்லாம் hundredths.

அதனால்

```text
300/100 + 20/100 + 5/100
= 325/100
```

## Lockhart சொல்ல வருவது

எந்த quantity-க்கும் ஒரே ஒரு பெயர் இல்லை.

உதாரணமாக

```text
3.25
=
3 + 2/10 + 5/100
=
325/100
=
13/4
=
3¼
=
3250/1000
```

இவை அனைத்தும் **ஒரே quantity**.

அதனால்

> "ஏன் நூறால் அளக்க வேண்டும்?"

என்றால் பதில்:

> **நூறால் அளக்க வேண்டிய அவசியமே இல்லை.**
>
> **அந்த நேரத்தில் கணக்கை எளிதாக்க எந்த unit வசதியாக இருக்கிறதோ, அந்த unit-ஐ தேர்ந்தெடுக்கிறோம்.**

இதுதான் fractions மற்றும் arithmetic-ன் மிக ஆழமான கருத்து:

> **Arithmetic என்பது quantity-யை மாற்றுவது அல்ல.**
>
> **அதே quantity-க்கு வசதியான பெயரை (representation) தேர்ந்தெடுப்பது.**

---------------------------------------------

``3 + 2/10 + 5/100`இப்படியே அழைக்கலாம்னு எதற்காக மேல் கூறியது போல் செய்ய வேண்டும்`


அப்படியே அழைக்கலாம். உண்மையில்

```text
3 + 2/10 + 5/100
```

என்பது தவறில்லை. அதுவே 3.25 என்பதன் உண்மையான அர்த்தம்.

அப்படியிருக்க எதற்கு

```text
300/100 + 20/100 + 5/100
= 325/100
```

என்று மாற்ற வேண்டும்?

காரணம் **கூட்டுவது எளிதாக இருக்க வேண்டும்** என்பதற்காக.

---

### பணம் உதாரணம்

உன்னிடம்

* ₹3
* 20 பைசா
* 5 பைசா

இருக்கிறது.

இப்படியே வைத்திருக்கலாம்:

```text
₹3 + 20p + 5p
```

இதில் எந்தத் தவறும் இல்லை.

ஆனால் எல்லாவற்றையும் பைசாவாக மாற்றினால்:

```text
300p + 20p + 5p
= 325p
```

இப்போது ஒரே unit (பைசா) என்பதால் நேராக கூட்ட முடிகிறது.

---

அதேபோல்

```text
3 + 2/10 + 5/100
```

இல்

* 3 என்பது wholes
* 2/10 என்பது tenths
* 5/100 என்பது hundredths

என்று மூன்று வேறு units உள்ளன.

அவற்றை ஒரே unit (hundredths) ஆக மாற்றினால்:

```text
3 = 300/100
2/10 = 20/100
5/100 = 5/100
```

ஆகவே

```text
300/100 + 20/100 + 5/100
= 325/100
```

இப்போது எல்லாமே hundredths என்பதால் numerator-களை மட்டும் கூட்டினால் போதும்:

```text
(300 + 20 + 5)/100
= 325/100
```

---

### முக்கியமான கருத்து

**மாற்ற வேண்டிய அவசியம் இல்லை.**

```text
3 + 2/10 + 5/100
```

என்பதும் சரி.

```text
325/100
```

என்பதும் சரி.

```text
13/4
```

என்பதும் சரி.

```text
3.25
```

என்பதும் சரி.

இவை எல்லாம் ஒரே quantity-க்கு வெவ்வேறு பெயர்கள்.

Lockhart சொல்ல வருவது:

> Arithmetic என்பது புதிய quantity உருவாக்குவது அல்ல; அதே quantity-க்கு கணக்கிட வசதியான representation-ஐ தேர்ந்தெடுப்பது.

அதனால்

```text
3 + 2/10 + 5/100
```

இப்படியே விட்டுவிடலாம்.

ஆனால் **கூட்ட, கழிக்க, ஒப்பிட, simplify செய்ய** வேண்டுமென்றால் ஒரே unit-ஆக மாற்றுவது மிகவும் இயல்பானது. அதனால்தான்

```text
300/100 + 20/100 + 5/100
```

என்று எழுதுகிறோம். இது quantity-யை மாற்றவில்லை; **பெயரை மட்டும் மாற்றுகிறது**.
