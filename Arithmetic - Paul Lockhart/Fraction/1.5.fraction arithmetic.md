அருமையான பகுதி இது. Paul Lockhart இங்கே உண்மையில் **fraction arithmetic** கற்றுக் கொடுக்கவில்லை; "fraction என்றால் என்ன?" என்ற ஆழமான கருத்தை உருவாக்குகிறார். ஒவ்வொரு கருத்தையும் ஆழமாகப் பார்ப்போம்.

---

# 1. Fraction என்பது புதிய வகை எண்

முன்பு நமக்கு

```
1,2,3,4,5,...
```

என்ற counting numbers மட்டும் இருந்தன.

இப்போது division காரணமாக

```
1/2
2/3
57/11
```

போன்ற புதிய எண்கள் வந்துவிட்டன.

உதாரணம்:

```
57/11
```

என்றால்,

* ஒன்றை 11 சம பாகங்களாக வெட்டு
* அந்தப் பாகங்களில் 57 எடுத்துக்கொள்

அவ்வளவுதான்.

---

# 2. ஒரே denominator இருந்தால் comparison மிகவும் எளிது

உதாரணம்:

```
9/11
7/11
```

இரண்டுமே elevenths.

அதாவது இரண்டிலும் piece size ஒன்றே.

அதனால்:

9 pieces > 7 pieces

எனவே

```
9/11 > 7/11
```

---

# 3. ஒரே numerator இருந்தாலும் comparison எளிது

உதாரணம்:

```
4/5
4/7
```

இரண்டிலும் 4 pieces இருக்கிறது.

ஆனால்:

ஒரு fifth > ஒரு seventh

ஏனெனில்

ஒரு பொருளை 5 பாகமாக வெட்டினால் கிடைக்கும் துண்டு,

7 பாகமாக வெட்டினால் கிடைக்கும் துண்டை விட பெரியது.

எனவே

```
4/5 > 4/7
```

---

# 4. ஆனால்

```
3/7
2/5
```

?

இங்கு பிரச்சினை:

* ஒன்று sevenths எண்ணுகிறது.
* மற்றொன்று fifths எண்ணுகிறது.

இது

> 3 apples vs 2 oranges

மாதிரி.

நேரடியாக compare செய்ய முடியாது.

---

# 5. Common language வேண்டும்

உதாரணம்:

ஒருவன் dozen-ஆக எண்ணுகிறான்.

```
2 dozen
```

மற்றவன் egg-ஆக எண்ணுகிறான்.

```
24 eggs
```

இருவரும் ஒரே quantity பற்றி பேசுகிறார்கள்.

ஆனால் language வேறு.

அதனால் எல்லாவற்றையும் egg ஆக மாற்றினால் comparison எளிது.

Fraction-ல அதைத்தான் common denominator என்பார்கள்.

---

# 6. Fraction என்பது multiplication கூட

Lockhart சொல்லும் மிக அழகான கருத்து:

```
3/7
```

என்பது

"3 sevenths"

அதாவது

```
3 × (1/7)
```

என்று பார்க்கலாம்.

7 என்பது group size.

3 என்பது copies.

---

# 7. ஏன்

```
1/3 = 2/6 = 3/9
```

?

ஏனெனில் piece size மாறுகிறது.

### முதலில்

ஒரு cake

```
|------|------|------|
   1/3
```

2 pieces எடுத்தால்

```
2/3
```

---

இப்போது ஒவ்வொரு third-யும் இரண்டு பாகமாக வெட்டு:

```
|---|---|---|---|---|---|
```

இப்போது

முன்னைய இரண்டு third-கள்

=

நான்கு sixths.

அதனால்

```
2/3 = 4/6
```

Quantity மாறவில்லை.

பெயர் மட்டும் மாறியது.

---

# 8. General rule

இரண்டு பக்கமும் ஒரே எண்ணால் பெருக்கினால் value மாறாது.

உதாரணம்:

```
2/3
```

இரண்டையும் 4-ஆல் பெருக்கு:

```
8/12
```

ஏன்?

ஏனெனில்

pieces நான்கு மடங்கு சிறியதாகிவிட்டன.

அதனால் pieces எண்ணிக்கையும் நான்கு மடங்கு அதிகரிக்க வேண்டும்.

---

# 9. Number enlarger (copy machine)

Lockhart-ன் favourite idea.

```
3/7
```

என்பதை

1-ஐ எடு.

முதலில்

7-ஆல் shrink செய்:

```
1 → 1/7
```

பிறகு

3 copies செய்:

```
3×1/7
=
3/7
```

அல்லது

முதலில்

1 → 3

பிறகு

7-ஆல் divide செய்:

```
3/7
```

அதே answer.

---

# 10. Common denominator

Compare

```
3/7
2/5
```

7 மற்றும் 5 language வேறு.

எனவே இரண்டையும் 35 language-க்கு மாற்றுவோம்.

```
3/7
=
15/35

2/5
=
14/35
```

இப்போது

pieces size ஒன்றே.

அதனால்

```
15 > 14
```

என்பது மட்டும் பார்க்க வேண்டும்.

அதனால்

```
3/7 > 2/5
```

---

# 11. ஏன் denominator product எடுக்கிறோம்?

```
7 × 5 = 35
```

ஏனெனில்

35 என்பது

7-க்கும் divide ஆகும்.

5-க்கும் divide ஆகும்.

அதனால் இரண்டையும் ஒரே language-ல் எழுத முடிகிறது.

---

# 12. Addition ஏன் easy ஆகிறது?

```
3/8 + 2/5
```

முதலில்

common denominator

40

```
3/8 = 15/40

2/5 = 16/40
```

இப்போது

piece size ஒன்றே.

அதனால்

வெறும் counting:

```
15 pieces +16 pieces

=31 pieces
```

அதாவது

```
31/40
```

# 15. Improper fraction

```
9/4
```

Mathematician language.

Lockhart இதையே விரும்புகிறார்.

ஏன்?

ஏனெனில்

```
2¼
```

என்றால்

whole, fraction என்று இரண்டு category.

ஆனால்

```
9/4
```

என்றால் எல்லாமே ஒரே வகை.

6 கூட அவர் விரும்பினால்

```
6/1
```

என்று எழுதுவார்.

எல்லா numbers-யும்

```
a/b
```

வடிவத்தில்.

ஒரே language.

# 16. Fraction-ஐ பல விதமாகப் பார்க்கலாம்

## Measurement

```
2/3
```

=

ஒன்றை மூன்று பாகமாக வெட்டி இரண்டு எடுத்தல்.

---

## Division

```
2÷3
```

---

## Ratio

```
2 : 3
```

அதுவே

```
2/3
```

# 17. 1½ × 6

இதன் உண்மையான அர்த்தம்

```
1½ copies of 6
```

ஒரு முழு 6

*

இன்னொரு அரை 6

```
6 +3
=9
```

---

அல்லது

```
1½
=
3/2
```

எனவே

```
3/2 × 6
```

என்பது

6-ஐ பாதியாக வெட்டு:

```
3
```

அதுபோன்ற மூன்று pieces:

```
3+3+3
=9
```

---

Lockhart முழுப் பகுதியில் சொல்லும் மையக் கருத்து:

> Fraction arithmetic என்பது புதிய விதி அல்ல.
>
> அது "எந்த unit-ல் எண்ணுகிறோம்?" என்ற language பிரச்சினை மட்டுமே.
>
> Common denominator என்பது language translation.
>
> Addition என்பது counting.
>
> Fraction என்பது ஒரு object அல்ல; அது ஒரு quantity-யை விவரிக்கும் பல பெயர்களில் ஒன்று.

இதிலிருந்து அடுத்த மிக அழகான பகுதி ஆரம்பிக்கிறது:

**Fraction multiplication மற்றும் division ஏன் numerator × numerator, denominator × denominator ஆகிறது?**
அதை Lockhart மிகவும் ஆழமாக "copy machine" மற்றும் "scaling" பார்வையில் விளக்குகிறார்.
