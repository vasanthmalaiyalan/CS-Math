அருமையான கேள்வி. இதுதான் Paul Lockhart சொல்ல விரும்பும் division-ன் உண்மையான படம். Long division algorithm-ஐ மறந்துவிட்டு, 248-ஐ உண்மையாக "பகிர்ந்து" பார்க்கலாம்.

நாம்

> **248 ÷ 7**

செய்கிறோம்.

அதாவது,

**248 பொருட்களை 7 பேருக்கு சமமாக கொடுக்க வேண்டும்.**

---

# படி 1 : 248 என்றால் என்ன?

248 என்பது

```text
2 hundreds + 4 tens + 8 ones
```

அதாவது

```text
200 + 40 + 8
```

---

# படி 2 : முதலில் hundreds-ஐ பகிர்வோமா?

2 hundreds இருக்கிறது.

7 பேருக்கு கொடுக்க வேண்டும்.

ஆனால்

```text
2 < 7
```

அதனால் ஒவ்வொருவருக்கும் ஒரு hundred கூட கொடுக்க முடியாது.

எனவே,

2 hundreds-ஐ உடைக்கிறோம்.

```text
2 hundreds
=
20 tens
```

இப்போது ஏற்கனவே இருந்த 4 tens சேர்த்து

```text
20 tens + 4 tens
=
24 tens
```

---

# படி 3 : 24 tens-ஐ 7 பேருக்கு

24 tens

=

21 tens + 3 tens

ஏனெனில்

```text
21 = 7×3
```

அதனால்

ஒவ்வொருவருக்கும்

```text
3 tens
```

அதாவது

30 கிடைக்கிறது.

மீதம்

```text
3 tens
```

---

# படி 4 : அந்த 3 tens-ஐ ones ஆக உடைப்போம்

3 tens

=

30 ones

ஏற்கனவே இருந்த 8 ones சேர்த்து

```text
30 + 8
=
38 ones
```

---

# படி 5 : 38 ones-ஐ 7 பேருக்கு

38 ones

=

35 ones + 3 ones

ஏனெனில்

```text
35 = 7×5
```

அதனால்

ஒவ்வொருவருக்கும்

5 ones.

மீதம்

3 ones.

---

# இப்போது மொத்தம்

ஒவ்வொருவருக்கும் கிடைத்தது

```text
3 tens + 5 ones
=
35
```

மீதம்

```text
3
```

எனவே

```text
248 ÷ 7 = 35 remainder 3
```

---

# இதை Lockhart பாணியில் எழுதினால்

```text
248

=
2 hundreds + 4 tens + 8 ones

=
24 tens + 8 ones

=
21 tens + 3 tens + 8 ones

=
7×3 tens + 38 ones

=
7×3 tens + 35 ones + 3 ones

=
7×(3 tens + 5 ones) + 3

=
7×35 + 3
```

அதனால்

```text
248 = 7×35 + 3
```

---

# இதுதான் long division algorithm உண்மையில் செய்கிறது

பள்ளியில் நாம்

```
      35
   -------
7 ) 248
   21
   ---
    38
    35
    ---
     3
```

என்று எழுதுகிறோம்.

ஆனால் அதன் பின்னால் நடப்பது:

```text
2 hundreds
↓ (உடைக்க)
24 tens
↓
21 tens + 3 tens
↓ (உடைக்க)
38 ones
↓
35 ones + 3 ones
```

என்பதே.

அதாவது **long division என்பது "digit இறக்குவது" அல்ல**.

அது

> **பெரிய units-ஐ சிறிய units-ஆக உடைத்து, அவற்றை சமமாக பகிர்ந்து, மீண்டும் regroup செய்வது**

என்பதே அதன் உண்மையான அர்த்தம்.

இதைத்தான் Egyptians counting coins கொண்டு செய்தார்கள்; நாம் Hindu-Arabic digits கொண்டு செய்கிறோம். Long division என்பது அந்த பழைய sharing process-ன் symbolic வடிவம் மட்டுமே.
