# 1. Division-க்கு இரண்டு அர்த்தங்கள்

56 jelly beans இருக்கிறது.

### கேள்வி 1

7 பேருக்கு சமமாகப் பிரித்தால் ஒவ்வொருவருக்கும் எவ்வளவு?

```text
56 ÷ 7 = 8
```

இங்கு:

* 7 = number of piles
* 8 = size of each pile

---

### கேள்வி 2

ஒவ்வொரு pile-லும் 8 jelly beans வைத்தால் எத்தனை piles உருவாகும்?

```text
56 ÷ 8 = 7
```

இங்கு:

* 8 = size of each pile
* 7 = number of piles

---

இரண்டும் வேறு கதைகள்.

ஆனால் mathematics-ல்:

```text
7×8=56
```

என்பதே உண்மை.

---

# 2. Multiplication symmetry காரணமாக division-க்கும் இரண்டு பார்வைகள்

### Partitive division

(Sharing)

56 chocolates

7 குழந்தைகள்

ஒவ்வொருவருக்கும்?

```text
56÷7=8
```

---

### Quotative division

(Grouping)

56 chocolates

ஒரு packet-ல் 8 chocolates

எத்தனை packets?

```text
56÷8=7
```

---

Lockhart சொல்கிறார்:

இவை interpretation மட்டும்.

எண்களுக்கு இதெல்லாம் தெரியாது.

அவற்றுக்கு

```text
7×8=56
```

அவ்வளவுதான்.

---

# 3. Modern view

Numbers care about behavior.

Not stories.

56÷8=7

ஏன்?

ஏனெனில்

```text
7×8=56
```

அவ்வளவு தான்.

"7 people", "8 jelly beans" என்பவை நம்முடைய interpretation.

# 5. Decimal point-ஐ மறந்துவிடலாம்

324.87 dollars

5 installments

கண்டுபிடிக்க வேண்டும்.

Lockhart சொல்கிறார்:

Decimal point பற்றி இப்போ கவலைப்படாதே.

324.87 dollars =

32487 cents

என்று நினைத்துக்கொள்.

---

இப்போது

32487÷5

---

6000 each

```text
5×6000=30000
```

மீதம்

2487

---

400 each

```text
5×400=2000
```

மீதம்

487

---

90 each

```text
5×90=450
```

மீதம்

37

---

7 each

```text
5×7=35
```

மீதம்

2

---

மொத்தம்:

```text
6000+400+90+7

=6497
```

மீதம் 2.

---

ஆனால் இது cents.

ஆகவே இரண்டு anti-shifts:

```text
6497
↓

64.97
```

மீதம்

2 cents.

---

அதாவது

ஒவ்வொரு installment:

```text
$64.97
```

மீதம்:

```text
$0.02
```

# 13. Golden rule

Lockhart-ன் முக்கியமான கருத்து:

> Don't be more accurate than your data.

Input approximate என்றால்

Output-ம் approximate.

# 14. Final question

Test tubes:

```text
30.25

27.12

32.62
```

கூட்டினால்:

```text
89.99 ml
```

ஆகிறது.

நீங்கள் measure செய்தபோது

"just over 90 ml"

என்று வந்தது.

Surprised ஆக வேண்டுமா?

இல்லை.

ஏனெனில்:

ஒவ்வொரு measurement-ம் itself approximate.

30.25 என்றால் அது exactly 30.250000000 அல்ல.

27.12 கூட exact இல்லை.

32.62 கூட exact இல்லை.

அவற்றின் small errors சேர்ந்து

89.99 → 90.01

ஆகலாம்.

அல்லது

89.98

ஆகலாம்.

அதனால்

"just over 90 ml"

என்பது perfectly reasonable.
