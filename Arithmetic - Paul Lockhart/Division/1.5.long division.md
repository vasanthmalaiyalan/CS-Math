# 4. Multiplication table இருவழியாக தெரிந்திருக்க வேண்டும்

நமக்கு தெரியும்:

```text
3×6=18
```

ஆனால் division-க்கு இது போதாது.

தெரிய வேண்டியது:

```text
18 = 3×6
```

அல்லது

```text
18÷6=3
18÷3=6
```

இவையும் உடனே தோன்ற வேண்டும்.

Lockhart சொல்வது:

Times table என்பது

ஒரு dictionary போல.

English → German

German → English

இரண்டும் தெரிய வேண்டும்.

# மிகவும் ஆழமான கருத்து

School-ல் நாம் நினைப்பது:

```text
      1346
13 )17503
```

என்று algorithm.

ஆனால் Lockhart பார்வையில்:

இது உண்மையில்

```text
17503 பொருட்கள்

13 பேருக்கு.

முதலில் 1000 each கொடு.

பிறகு 300 each கொடு.

பிறகு 40 each கொடு.

பிறகு 4 each கொடு.

பிறகு 2 each கொடு.

இன்னும் 5 பொருட்கள் மீதி.
```

Long division என்பது

**sharing process + estimation + repeated subtraction**

இவற்றை compact-ஆக எழுதும் notation மட்டுமே.

அதாவது

> Division algorithm என்பது mathematics அல்ல.

> Mathematics என்பது "எவ்வளவு safely கொடுக்கலாம்?" என்று தொடர்ந்து யோசிப்பதே. Algorithm என்பது அதன் shorthand மட்டும்.
