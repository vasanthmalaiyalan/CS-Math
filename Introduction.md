# அறிமுகம் (Introduction)

இந்த புத்தகம் **கணிதத்தை பயன்படுத்தி Computer Science-ல் வரும் பிரச்சனைகளை எப்படி புரிந்து கொண்டு தீர்ப்பது** என்பதை கற்றுக்கொடுக்கிறது.

உதாரணமாக:

* ஒரு algorithm சரியாக வேலை செய்கிறதா?
* ஒரு program எப்போதும் crash ஆகாமல் இயங்குமா?
* ஒரு security protocol பாதுகாப்பானதா?
* ஒரு network system சரியாக data அனுப்புமா?

இவற்றை கணித முறைகள் மூலம் நிரூபிக்கலாம்.

---

# ஏன் Proof (நிரூபணம்) முக்கியம்?

ஆசிரியர்கள் சொல்வது:

> "உண்மையான புரிதல் (Understanding) பெற Proof மிகவும் அவசியம்."

உதாரணம்:

நீ ஒரு algorithm O(log n) என்று மனப்பாடம் செய்யலாம்.

ஆனால்:

**ஏன் O(log n)?**

என்பதை Proof மூலம் காட்டினால் தான் உண்மையாக புரியும்.

# Testing மட்டும் போதாதா?

பலர் நினைப்பது:

> Program-ஐ ஆயிரம் முறை test செய்தால் சரி.

ஆனால் ஆசிரியர் சொல்வது:

இல்லை.

ஏனெனில்:

ஒரு bug

* 1000 test-ல் வராமல் இருக்கலாம்
* 1,000,001-வது input-ல் வரலாம்

Testing:

```
பல case-களை சோதிக்கும்
```

Proof:

```
எல்லா possible case-களுக்கும் சரி என்பதை நிரூபிக்கும்
```

உதாரணம்:

Password validation function.

```

```

```

```

# René Descartes

René Descartes என்ற புகழ்பெற்ற கணிதவியலாளர் மற்றும் தத்துவஞானி ஒரு பிரபலமான வாக்கியம் கூறினார்:

> Cogito ergo sum

Latin மொழி.

தமிழில்:

> "நான் சிந்திக்கிறேன்; ஆகவே நான் இருக்கிறேன்."

அல்லது

> "நான் யோசிக்கிறேன், எனவே நான் இருப்பது உறுதி."

---

அவர் சொல்வது:

நான் எல்லாவற்றையும் சந்தேகப்படலாம்.

ஆனால்

நான் இப்போது சந்தேகப்படுகிறேன் என்பதே

நான் சிந்திக்கிறேன் என்பதற்கான ஆதாரம்.

நான் சிந்திக்கிறேன் என்றால்

நான் இருக்கிறேன்.

---

அதாவது:

```
Thinking exists
↓
Thinker exists
↓
Therefore I exist
```

என்ற தத்துவ Proof.

# Footnote (கீழ்க்குறிப்பு) என்ன சொல்கிறது?

இது மிகவும் முக்கியமான கருத்து.

உண்மையில் Experiment மூலம்

ஒரு theory தவறு என்பதை நிரூபிக்கலாம்.

---

உதாரணம்:

Theory:

```
எல்லா காகங்களும் கருப்பு நிறம்.
```

நீ ஒரு வெள்ளை காகம் கண்டால்?

Theory உடனே தவறு.

---

ஆனால்:

10 லட்சம் கருப்பு காகங்களை பார்த்தாலும்

```
அடுத்த காகமும் கருப்பாகத்தான் இருக்கும்
```

என்று 100% நிரூபிக்க முடியாது.

---

அதனால் விஞ்ஞானிகள் பொதுவாக:

"இது உண்மை"

என்று சொல்ல மாட்டார்கள்.

அதற்கு பதில்:

> "இந்த theory இதுவரை நடந்த அனைத்து experiment-களையும் சரியாக predict செய்கிறது."

என்று சொல்வார்கள்.

# Computer Science-க்கு முக்கியமான கருத்து

இந்த Introduction முழுவதும் சொல்ல வருவது:

1. Computer Science என்பது coding மட்டும் அல்ல.
2. Logic மற்றும் Mathematics அதன் அடித்தளம்.
3. Proof இல்லாமல் முழுமையான நம்பிக்கை கிடையாது.
4. Testing பல case-களை பார்க்கும்.
5. Proof எல்லா case-களுக்கும் சரி என்பதை நிரூபிக்கும்.
6. அதனால் Algorithms, Security, Cryptography, Distributed Systems, Operating Systems போன்ற துறைகளில் Proof மிகவும் முக்கியம்.

ஒரு வரியில்:

> **"Computer Science-ஐ ஆழமாகப் புரிந்துகொள்ள Coding மட்டும் போதாது; Logic, Mathematics, மற்றும் Proof-களைப் புரிந்துகொள்ள வேண்டும்"** என்பதைத்தான் இந்த Introduction சொல்லுகிறது.
