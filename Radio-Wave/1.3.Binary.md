``பிறகு அவை Binary-ஆக மாறும்.

01111101
10000010
01111111
10001100
...
இவை "வ" என்ற எழுத்து அல்ல.
இவை அந்த நொடியில் உங்கள் குரலின் உயரம் (Pitch), சத்தம் (Volume), அதிர்வெண் (Frequency) போன்றவற்றின் அளவீடுகள் மட்டுமே.`இந்த டேட்டா வைத்து மட்டும் அது எப்படி நான் பேசிக் கொண்டிருப்பதை எதுத்தாப்பில் அதைக் கேட்டுக் கொண்டிருக்கும் போனில் அதைக் கேட்டுக் கொண்டிருக்கும் நபருக்கு அப்படியே எப்படி அது சென்று சேருகிறது`

இந்தக் கேள்விதான் டிஜிட்டல் தொடர்பின் (Digital Communication) இதயம். நீங்கள் சரியான இடத்தில் சந்தேகம் கேட்கிறீர்கள்.

நீங்கள் நினைப்பது:

> **"0 மற்றும் 1-ல் 'வணக்கம்' என்று எழுதப்படவில்லையே! அப்படியிருக்க, மறுபக்கத்தில் இருப்பவர் நான் பேசிய அதே குரலை எப்படி கேட்கிறார்?"**

பதில்: **ஏனென்றால் அந்த 0 மற்றும் 1 "வார்த்தைகளை" சேமிக்கவில்லை; "ஒலியின் வடிவத்தை (Waveform)" சேமிக்கிறது.**

ஒரு உதாரணம் பார்ப்போம்.

## நீங்கள் "ஆ..." என்று ஒரு வினாடி நீளமாகச் சொல்கிறீர்கள்.

அந்த ஒலி ஒரு அலை போல இருக்கும்.

```
      /\      /\      /\
     /  \    /  \    /  \
____/    \__/    \__/    \____
```

இப்போது ADC என்ன செய்கிறது?

இந்த அலையை ஒரு வினாடிக்கு **48,000 முறை** (அல்லது 44,100 முறை) அளவிடுகிறது.

உதாரணமாக:

```
நேரம்      அளவு
0 ms       120
0.02 ms    125
0.04 ms    130
0.06 ms    128
0.08 ms    122
...
```

இந்த எண்கள்தான் Binary ஆக மாறுகின்றன.

```
120 → 01111000
125 → 01111101
130 → 10000010
128 → 10000000
122 → 01111010
```

## மறுபக்கத்தில் என்ன நடக்கிறது?

அமெரிக்காவில் இருக்கும் போனுக்கு இந்த எண்கள் கிடைக்கின்றன.

அது என்ன செய்கிறது?

அதே வரிசையில்:

```
120
125
130
128
122
...
```

என்று வாசிக்கிறது.

பிறகு ஸ்பீக்கரிடம் சொல்கிறது:

* "முதல் 0.02 ms-ல் இவ்வளவு வெளியே வா."
* "அடுத்த 0.02 ms-ல் இன்னும் கொஞ்சம் வெளியே வா."
* "அடுத்தது கொஞ்சம் உள்ளே போ."
* "அடுத்தது மீண்டும் வெளியே வா."

ஸ்பீக்கர் இந்த கட்டளைகளை ஒரு வினாடிக்கு **48,000 முறை** செய்கிறது.

அதனால் மீண்டும் அதே ஒலி அலை உருவாகிறது.

```
      /\      /\      /\
     /  \    /  \    /  \
____/    \__/    \__/    \____
```

இதுதான் உங்கள் காதுக்குச் சென்று **"ஆ..."** என்று கேட்கிறது.

## ஒரு எளிய ஒப்புமை

நீங்கள் ஒரு மலையைப் பார்த்து அதன் உயரத்தை ஒவ்வொரு ஒரு மீட்டருக்கும் எழுதுகிறீர்கள்.

```
100 m
105 m
110 m
108 m
102 m
...
```

இந்த பட்டியலை வேறொருவரிடம் கொடுத்தால், அவர் அந்த எண்களை வைத்து **அதே மலையின் வடிவத்தை** மீண்டும் வரைய முடியும்.

அதேபோல்:

* மைக்ரோஃபோன் → ஒலி அலையின் "உயரங்களை" (அழுத்த அளவுகளை) பதிவு செய்கிறது.
* இணையம் → அந்த எண்களை அனுப்புகிறது.
* ஸ்பீக்கர் → அந்த எண்களைப் பயன்படுத்தி **அதே ஒலி அலையை மீண்டும் உருவாக்குகிறது.**

**அதனால் போன்கள் "உங்கள் வார்த்தைகளை" அனுப்புவதில்லை; "உங்கள் குரலின் ஒலி அலையின் வரைபடத்தை" அனுப்புகின்றன.**

அதனால்தான் உங்கள் குரலின் தனித்துவம், உணர்ச்சி, சிரிப்பு, அழுகை, கோபம் போன்றவை கூட மறுபக்கத்தில் இருக்கும் நபருக்கு கிட்டத்தட்ட அதேபோலக் கேட்கின்றன. இது ஒலி அலையின் வடிவம் மிகத் துல்லியமாக மீண்டும் உருவாக்கப்படுவதால்தான்.

---------------------------------------------------

`அதாவது நான் பேசுவதை ஒரு மெல்லிய தகடால் அசைவை வைத்து கணிக்க முடிந்தால் அது என்னவென்று உலகத்தில் எந்த மூலையில் இருந்தாலும் தெரிந்து கொள்ள முடியுமா`

ஆம். **கொள்கை ரீதியாக அதுதான்.** ஆனால் ஒரு சிறிய திருத்தம் இருக்கிறது.

நீங்கள் சொன்னது:

> "ஒரு மெல்லிய தகடால் அசைவை வைத்து கணிக்க முடிந்தால்..."

உண்மையில் அது **கணிப்பது இல்லை**. அது **அளவிடுகிறது (Measures)**.

இதை ஒரு உதாரணத்துடன் பார்ப்போம்.

நான் ஒரு காகிதத்தில் உங்கள் குரலின் ஒலி அலையை மிகவும் துல்லியமாக வரைந்து வைத்திருக்கிறேன் என்று நினைத்துக் கொள்ளுங்கள்.

அந்த வரைபடத்தை:

* இந்தியாவில் இருக்கும் ஒருவர் பார்த்தாலும்,
* அமெரிக்காவில் இருக்கும் ஒருவர் பார்த்தாலும்,
* ஜப்பானில் இருக்கும் ஒருவர் பார்த்தாலும்,

அனைவரும் **அதே ஒலி அலையை** மீண்டும் உருவாக்க முடியும்.

ஏன்?

ஏனெனில் இயற்பியல் விதிகள் உலகம் முழுவதும் ஒன்றுதான்.

---

### அதனால்தான்...

மைக்ரோஃபோன் என்ன செய்கிறது?

* "இவர் தமிழ் பேசுகிறார்" என்று அதற்குத் தெரியாது.
* "இவர் கோபமாக பேசுகிறார்" என்றும் தெரியாது.
* "இவர் 'வணக்கம்' சொன்னார்" என்றும் தெரியாது.

அது வெறும்:

* இந்த நொடியில் காற்றழுத்தம் இவ்வளவு.
* அடுத்த நொடியில் இவ்வளவு.
* அடுத்த நொடியில் இவ்வளவு.

என்று **அளவிடுகிறது.**

இந்த அளவீடுகள் உலகத்தின் எந்த மூலையிலும் ஒரே மாதிரியாக இருக்கும்.

---

### ஒரு ஒப்புமை

நீங்கள் உங்கள் இதயத் துடிப்பை **ECG** இயந்திரத்தில் பதிவு செய்கிறீர்கள்.

அந்த ECG காகிதத்தை இந்தியாவில் உள்ள மருத்துவர் பார்த்தாலும், அமெரிக்காவில் உள்ள மருத்துவர் பார்த்தாலும், இருவரும் **அதே இதயத் துடிப்பை** புரிந்துகொள்வார்கள்.

ஏனெனில் ECG காகிதம் இதயத்தை "கணிக்கவில்லை". அது இதயத்தின் செயல்பாட்டை **பதிவு** செய்திருக்கிறது.

மைக்ரோஃபோனும் அதே மாதிரி.

---

### அதனால் உங்கள் வாக்கியத்தை இன்னும் துல்லியமாகச் சொன்னால்:

> **"நான் பேசும்போது உருவாகும் ஒலி அலையை மைக்ரோஃபோன் மிகத் துல்லியமாக அளவிட்டு பதிவு செய்கிறது. அந்த பதிவை உலகின் எந்த மூலையிலும் இருக்கும் ஒரு சாதனம் பெற்றுக்கொண்டால், அதே ஒலி அலையை மீண்டும் உருவாக்க முடியும்."**

இதுதான் தொலைபேசி, WhatsApp Call, Zoom, Google Meet போன்ற அனைத்தும் இயங்கும் அடிப்படை அறிவியல்.

உண்மையில், இந்தக் கேள்விகளின் மூலம் நீங்கள் **தகவல் (Information) என்றால் என்ன?** என்ற கணினி அறிவியலின் மிக அடிப்படை கருத்தை நோக்கி செல்கிறீர்கள். இந்தக் கருத்தைப் புரிந்துகொண்டால், மைக்ரோஃபோன், ஸ்பீக்கர், இணையம், டிஜிட்டல் தொடர்பு ஆகிய அனைத்தும் ஒரே சங்கிலியாக எப்படி வேலை செய்கின்றன என்பது மிகவும் தெளிவாகிவிடும்.
